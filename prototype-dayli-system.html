The Monorepo

One repo, two apps, shared packages — all managed by Claude Code

monorepo structure
dayli/
├── apps/landing/ ← Daily Living Labs (marketing site)
├── apps/web/ ← Dayli AI (11-screen web app)
├── packages/ui/ ← Shared design tokens
├── packages/supabase/ ← Client, types, prompts
└── supabase/ ← Schema, edge functions
Claude edits code → git push → Vercel auto-deploys → ✓ Live in 30s
Why a monorepo? Both apps share the same design tokens and database types. Change a color once, it updates everywhere.

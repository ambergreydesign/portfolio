The Stack

We use Claude Code as our development engine connected to GitHub, Vercel, Figma, and Supabase. Here's how the system works and why it matters for our team now, and new teams in the future.

Architecture

Everything runs through Claude Code. Claude writes and edits our codebase.

Architecture

Our codebase lives in a single monorepo

Architecture

The monorepo connects to two apps through Vercel — the Daily Living Labs landing page and the main Dayli AI web app.

Deploy

When we push code to GitHub, Vercel automatically deploys it live within 30 seconds.

Backend

Supabase handles our database, authentication, and AI chat functions.

Design

And Figma is where we can get hands on with our design.

Onboarding

The repo has a step by step visual setup, and a built-in slash command — /setup. Once you're in Claude Code, type /setup and Claude walks you through the same checklist conversationally. It runs each check for you, tells you what passed and what needs fixing, and guides you through the fix in plain language.

Daily Workflow

For development work, you start a chat and tell Claude which app you're working on — the landing page or the web app. Describe the changes, Claude codes, push to git, and it's live.

Memory

Each session is fresh, but Claude picks up the full project context automatically by reading the project rules, memory files, and codebase. What carries between sessions is the code in git and the memory files, not the conversation itself.

Memory

Each Claude session has persistent memory. It knows the project structure, the design system, the deployment workflow, and the team preferences. You don't have to re-explain context between sessions.

Why It Works

There are three main reasons for this structure.

Why · Speed

#1: Speed. A design change goes from idea to live, in under two minutes. No build pipeline to babysit, no staging environment to maintain.

Why · Accessibility

#2: Accessibility for non-engineers. You don't need to know React or Tailwind to ship changes. Describe what you want in plain language, review the result, push it live.

Why · Scalability

#3: Scalability for future teams. The monorepo structure, shared design tokens, and Supabase backend mean a new developer, or a new Claude chat, can pick up any part of the system without breaking the rest.

That's the system. Three tool connections, two setup guides to get your machine ready, one entry point to keep everyone aligned, and a workflow that lets anyone on the team iterate at the speed of conversation.
